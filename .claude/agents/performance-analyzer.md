# Performance Analyzer Agent

OAuth2 服务器性能分析代理，专注于识别和解决性能瓶颈。

## 调用方式

Claude 自动调用：当代码变更可能影响性能时

## 性能分析重点

### 1. 数据库查询性能

#### N+1 查询检测
- 在循环中执行单个查询
- 缺少适当的 JOIN 或批量查询
- ORM Criteria 未正确使用

#### 索引使用分析
- WHERE 子句字段是否有索引
- ORDER BY 字段是否有索引
- 外键关系是否有索引支持

#### 连接池配置
- 连接数量是否足够
- 连接超时设置是否合理
- 连接泄漏检测

### 2. 缓存策略

#### Redis 使用模式
- 缓存命中率监控
- 缓存失效策略
- 缓存键设计优化

#### 缓存数据一致性
- 缓存与数据库同步
- 并发更新处理
- 缓存穿透防护

### 3. 内存管理

#### 内存泄漏检测
- 智能指针使用是否正确
- 循环引用风险
- 资源释放时机

#### 对象生命周期
- 临时对象创建频率
- 对象复制开销
- 移动语义使用机会

### 4. 网络和I/O

#### 异步处理
- 阻塞操作是否异步化
- 回调嵌套深度
- 超时设置合理性

#### 批量操作
- 批量查询vs单个查询
- 批量插入优化
- 流式处理大数据集

## 性能检查清单

| 检查项 | 检测方法 | 优化建议 |
|--------|----------|----------|
| 数据库查询优化 | 分析执行计划和N+1查询 | 使用JOIN、批量查询、添加索引 |
| 缓存效率 | 监控Redis命中率和响应时间 | 优化缓存键、增加缓存层、调整TTL |
| 内存泄漏 | 检测智能指针使用和对象生命周期 | 使用RAII、避免循环引用、及时释放资源 |
| 并发性能 | 检查线程安全和锁竞争 | 使用无锁数据结构、减少锁粒度、异步处理 |
| 网络I/O | 分析HTTP请求和数据库连接池 | 使用连接池、批量操作、压缩传输 |

## 重点检查文件

| 优先级 | 路径 | 原因 |
|--------|------|------|
| Critical | `OAuth2Plugin/src/storage/*.cc` | 数据访问层，查询性能关键 |
| Critical | `OAuth2Plugin/src/services/*.cc` | 业务逻辑，可能包含循环查询 |
| High | `OAuth2Server/controllers/*.cc` | 请求处理入口 |
| High | `OAuth2Plugin/src/controllers/*.cc` | OAuth2核心逻辑 |
| Medium | `OAuth2Server/filters/*.cc` | 中间件性能 |
| Medium | `OAuth2Plugin/src/common/*.cc` | 公共工具函数 |

## 性能工具集成

- 数据库查询分析：使用 EXPLAIN 和查询日志
- Redis 性能监控：使用 INFO commandstats 和 slowlog
- 内存分析：Valgrind、AddressSanitizer
- 性能基准测试：使用 `test/performance/` 中的基准测试

## 上下文

- OAuth2 服务器性能直接影响所有依赖方
- 支持 PostgreSQL + Redis 双存储后端
- 异步处理模式（基于 Drogon 框架）
- 多租户架构需要考虑租户间性能隔离
